# 04 — نظام ERP المتكامل

> قلب المشروع. نظام موحّد يربط المخزون والطلبات والمشتريات والمحاسبة وإدارة العملاء بحيث تكون كل البيانات متزامنة تلقائياً.

---

## 1. لوحة المعلومات (Dashboard)

نظرة سريعة شاملة لحظة فتح اللوحة:

- إجمالي المبيعات (اليوم / الأسبوع / الشهر).
- عدد الطلبات حسب الحالة (جديد، قيد التجهيز، مشحون، مُسلّم).
- تنبيهات المخزون المنخفض.
- أحدث الطلبات.
- المنتجات الأكثر مبيعاً.
- رسوم بيانية للمبيعات والأرباح.
- عدد العملاء الجدد.

---

## 2. إدارة المنتجات (Catalog Management)

### العمليات الأساسية
- إضافة / تعديل / حذف منتج.
- رفع صور متعددة لكل منتج (ورفع بالجملة).
- تعيين التصنيف والنوع (روسي / صيني).
- إدخال السعر والسعر قبل الخصم.

### المتغيرات (Variants)
- متغيرات حسب المقاس واللون.
- لكل متغير: SKU خاص، سعر (إن اختلف)، كمية مستقلة.

### التنظيم
- التصنيفات والتصنيفات الفرعية.
- الوسوم (Tags) للتجميع والفلترة.
- منتجات مميزة / جديدة / الأكثر مبيعاً.

### الاستيراد والتصدير
- استيراد منتجات من ملف Excel/CSV.
- تصدير قائمة المنتجات.
- تحديث الأسعار والكميات بالجملة.

---

## 3. إدارة المخزون (Inventory Management)

### التتبّع اللحظي
- كمية متاحة لحظية لكل SKU.
- خصم تلقائي عند تأكيد الطلب.
- إعادة الكمية تلقائياً عند الإلغاء أو الإرجاع.

### الحجز المؤقت (Reservation)
- حجز الكمية مؤقتاً عند إنشاء الطلب لتجنّب البيع المزدوج.
- تحرير الحجز إذا لم يُكمل الطلب خلال مدة محددة.

### التنبيهات
- تنبيه عند وصول الكمية للحد الأدنى (قابل للتعديل لكل منتج).
- تقرير المنتجات النافدة أو شبه النافدة.

### المستودعات المتعددة (قابل للتفعيل)
- إدارة أكثر من مستودع/فرع.
- توزيع الكميات بين المستودعات.
- تحويل المخزون بين المستودعات.

### حركة المخزون (Stock Movements)
| نوع الحركة | الوصف |
|------------|-------|
| إدخال | استلام بضاعة جديدة (شراء) |
| إخراج | بيع / صرف |
| مرتجع | إرجاع من عميل |
| تالف | إتلاف بضاعة |
| تسوية | تعديل بعد الجرد |
| تحويل | بين المستودعات |

### الجرد (Stock Count)
- جرد دوري للمخزون.
- مقارنة الكمية الفعلية بالنظام.
- تسوية الفروقات مع تسجيل السبب.

---

## 4. إدارة الطلبات (Order Management)

### عرض الطلبات
- قائمة موحّدة بكل الطلبات قابلة للفلترة (الحالة، التاريخ، العميل، طريقة الدفع).
- بحث برقم الطلب أو اسم/هاتف العميل.

### حالات الطلب
```
جديد → مؤكد → قيد التجهيز → مشحون → مُسلّم
                                  ↘ ملغي
                                  ↘ مرتجع
```

### العمليات على الطلب
- تأكيد / إلغاء الطلب.
- تعديل الطلب (إضافة/حذف منتج، تعديل كمية).
- تعيين شركة الشحن والمندوب.
- طباعة الفاتورة وبوليصة الشحن.
- إضافة ملاحظات داخلية.
- إرسال إشعار للعميل عند تغيّر الحالة.

### المرتجعات والاستبدال (Returns)
- طلب إرجاع/استبدال.
- الموافقة أو الرفض.
- إعادة المبلغ (Refund) وإعادة الكمية للمخزون.
- تتبّع حالة المرتجع.

---

## 5. المشتريات والموردين (Procurement)

### الموردون
- تسجيل بيانات الموردين (الاسم، التواصل، الشروط).
- سجل التعاملات لكل مورد.

### أوامر الشراء (Purchase Orders)
- إنشاء أمر شراء.
- تتبّع حالة الأمر (معلّق، مستلم جزئياً، مستلم بالكامل).
- تحديث المخزون تلقائياً عند الاستلام.
- تسجيل تكلفة الشراء.

### تحليل التكلفة
- مقارنة سعر التكلفة بسعر البيع.
- حساب هامش الربح لكل منتج.

---

## 6. المحاسبة والتقارير (Accounting & Reports)

### الفواتير
- إنشاء فاتورة تلقائية لكل طلب.
- طباعة وتصدير PDF.

### التقارير المتاحة
| التقرير | المحتوى |
|---------|---------|
| المبيعات | يومي / أسبوعي / شهري / سنوي |
| الأرباح | الإيرادات مقابل التكاليف |
| المخزون | الكميات وقيمتها |
| المنتجات | الأكثر/الأقل مبيعاً |
| العملاء | الأكثر شراءً، الجدد |
| المشتريات | حسب المورد والفترة |
| الكوبونات | استخدام أكواد الخصم |
| المرتجعات | معدّل الإرجاع |

### التصدير
- تصدير كل التقارير إلى Excel و PDF.
- جدولة تقارير دورية (اختياري).

---

## 7. إدارة العملاء (CRM)

### ملف العميل
- البيانات الشخصية والتواصل.
- سجل الطلبات والقيمة الإجمالية للمشتريات.
- العناوين المحفوظة.
- الملاحظات.

### التقسيم (Segmentation)
- عملاء جدد / متكررون / VIP.
- عملاء غير نشطين (لإعادة استهدافهم).

### التسويق
- حملات بريدية / SMS موجّهة.
- كوبونات مخصصة لشرائح معينة.
- برنامج ولاء (مرحلة لاحقة محتملة).

---

## 8. الإعدادات العامة

- إعدادات المتجر (الاسم، الشعار، التواصل).
- إعدادات الشحن (المناطق والرسوم).
- إعدادات الدفع (البوابات المفعّلة).
- إعدادات الضرائب (إن وُجدت).
- إدارة الصفحات الثابتة (سياسات، من نحن).
- إدارة البانرات والعروض.
- إدارة اللغات والعملة.

---

## 9. الترابط التلقائي بين الوحدات

```
بيع (طلب) ──► خصم المخزون ──► تسجيل محاسبي ──► تحديث ملف العميل
شراء ──────► زيادة المخزون ──► تسجيل تكلفة ──► تحديث ملف المورد
إرجاع ─────► إعادة المخزون ──► استرداد مالي ──► تحديث سجل العميل
```

> هذا الترابط هو ما يميّز نظام ERP عن مجرد متجر — كل عملية تنعكس تلقائياً على باقي الوحدات دون إدخال يدوي مكرر.
