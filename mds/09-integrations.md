# 09 — التكاملات الخارجية (Integrations)

## 1. بوابات الدفع (Payment)

### طرق الدفع المطلوبة
| الطريقة | الوصف |
|---------|-------|
| **الدفع عند الاستلام (COD)** | الأكثر استخداماً محلياً — لا يحتاج بوابة |
| **الدفع الإلكتروني بالبطاقة** | فيزا / ماستركارد عبر بوابة محلية |
| **المحافظ الإلكترونية** | حسب المتوفر محلياً |

### اعتبارات اختيار البوابة
- دعم السوق المحلي والعملة المحلية.
- رسوم المعاملات المعقولة.
- سهولة التكامل (API موثّق).
- الأمان والالتزام بمعايير PCI-DSS.
- دعم الاسترداد (Refund) عبر API.

### آلية العمل
```
العميل يختار الدفع الإلكتروني
   → تحويل لصفحة البوابة الآمنة
   → إتمام الدفع
   → استقبال نتيجة الدفع (Webhook)
   → تأكيد الطلب تلقائياً عند نجاح الدفع
```

> **مهم:** بيانات البطاقة لا تمر عبر خوادم المتجر إطلاقاً — تُعالَج مباشرة لدى البوابة.

## 2. شركات الشحن (Shipping)

### الوظائف المطلوبة
- حساب رسوم الشحن حسب المنطقة/المدينة.
- إنشاء شحنة وبوليصة شحن.
- توليد رقم تتبّع.
- تحديث حالة الشحنة تلقائياً.

### آلية التكامل
| الطريقة | الوصف |
|---------|-------|
| تكامل API مباشر | ربط مع شركة شحن تدعم API |
| إدخال يدوي | تسجيل رقم التتبّع يدوياً (للبداية) |

### إعدادات الشحن في النظام
- تعريف مناطق الشحن ورسومها.
- شحن مجاني فوق مبلغ معيّن (اختياري).
- مدة التوصيل التقديرية لكل منطقة.

---

## 3. الإشعارات (Notifications)

### القنوات
| القناة | الاستخدام |
|--------|-----------|
| **البريد الإلكتروني** | تأكيد الطلب، الفواتير، التحديثات |
| **الرسائل النصية (SMS)** | تأكيد الطلب، رقم التتبّع، OTP |
| **إشعارات الموقع** | تحديثات فورية للمستخدم المسجّل |
| **واتساب** (اختياري) | تأكيدات وتحديثات عبر واتساب للأعمال |

### أنواع الإشعارات
- تأكيد التسجيل وكلمة المرور المؤقتة.
- رمز التحقق (OTP).
- تأكيد الطلب.
- تحديثات حالة الطلب.
- إشعارات العروض والكوبونات (تسويقي).
- تنبيهات إدارية (مخزون منخفض، طلب جديد).

## 4. خدمات أخرى

### تحليلات الموقع (Analytics)
- Google Analytics لتتبّع الزوار والسلوك.
- تتبّع معدّل التحويل (Conversion Rate).
- تتبّع السلات المتروكة (Abandoned Carts).

### تسجيل الدخول الاجتماعي (اختياري)
- تسجيل الدخول عبر Google.
- تسجيل الدخول عبر Facebook.

### الدعم والمحادثة
- أداة شات مباشر (Live Chat).
- زر واتساب عائم.

### البريد التسويقي
- ربط مع خدمة نشرة بريدية (Email Marketing).
- حملات بريدية مجدوَلة.

## 5. واجهات برمجية داخلية (Internal APIs)

النظام يوفّر واجهات داخلية تربط المتجر بالـ ERP:

| الواجهة | الوظيفة |
|---------|---------|
| Products API | جلب المنتجات والمخزون للمتجر |
| Orders API | إنشاء واستقبال الطلبات |
| Inventory API | تحديث وحجز المخزون |
| Customers API | إدارة بيانات العملاء |
| Reports API | جلب بيانات التقارير |

---

## 6. اعتبارات التكامل

- **معالجة الفشل**: إعادة المحاولة التلقائية عند فشل الاتصال بخدمة خارجية.
- **الـ Webhooks**: استقبال تحديثات الدفع والشحن بشكل آمن (التحقق من التوقيع).
- **السجلّات**: تسجيل كل التعاملات مع الخدمات الخارجية للمراجعة.
- **البيئة التجريبية**: اختبار كل التكاملات في بيئة Sandbox قبل التشغيل الفعلي.
- **المرونة**: تصميم يسمح بتبديل مزوّد الخدمة (دفع/شحن) دون إعادة بناء كبيرة.
