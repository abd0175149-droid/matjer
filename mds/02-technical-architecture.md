# 02 — البنية التقنية (Technical Architecture)

## 1. نمط المعمارية

النظام يُبنى وفق معمارية **متعددة الطبقات (Layered) مع فصل الواجهة عن الخادم** عبر API. هذا يسمح بتطوير المتجر ولوحة الـ ERP بشكل مستقل مع مشاركة نفس الخادم والبيانات.

```
┌──────────────────────────────────────────────────────┐
│                  طبقة العرض (Presentation)            │
│   متجر العميل (Next.js)  +  لوحة ERP (React)         │
└─────────────────────────┬────────────────────────────┘
                          │ REST API / GraphQL
┌─────────────────────────▼────────────────────────────┐
│                طبقة منطق العمل (Business Logic)        │
│   خدمات: المنتجات · الطلبات · المخزون · المحاسبة      │
└─────────────────────────┬────────────────────────────┘
                          │
┌─────────────────────────▼────────────────────────────┐
│                طبقة البيانات (Data Layer)             │
│   PostgreSQL · Redis · تخزين الصور (S3/Cloudinary)   │
└──────────────────────────────────────────────────────┘
```

---

## 2. التقنيات المقترحة (Tech Stack)

### الواجهة الأمامية — المتجر
| العنصر | الخيار | السبب |
|--------|--------|-------|
| الإطار | **Next.js (React)** | أداء عالٍ + دعم SEO ممتاز (SSR/SSG) |
| التنسيق | Tailwind CSS | تطوير سريع وتصميم متجاوب |
| إدارة الحالة | Redux Toolkit / Zustand | إدارة السلة والجلسة |

### الواجهة الإدارية — لوحة ERP
| العنصر | الخيار |
|--------|--------|
| الإطار | React + مكتبة لوحات (مثل Refine / Ant Design Pro) |
| الجداول والتقارير | AG Grid / Recharts |

### الخادم (Backend)
| العنصر | الخيار | بديل |
|--------|--------|------|
| البيئة | **Node.js + NestJS** | Laravel (PHP) |
| نمط الـ API | REST (أو GraphQL) | — |
| المصادقة | JWT + Refresh Tokens | — |

### قاعدة البيانات والتخزين
| العنصر | الخيار | الاستخدام |
|--------|--------|-----------|
| القاعدة الرئيسية | **PostgreSQL** | كل البيانات العلائقية |
| التخزين المؤقت | **Redis** | الجلسات، السلة، تسريع الاستعلامات |
| البحث | Meilisearch / Elasticsearch | بحث وفلترة سريعة |
| الصور والملفات | S3 / Cloudinary + CDN | تخزين وعرض سريع للصور |

## 3. خيار بديل سريع للإطلاق

بدلاً من البناء من الصفر، يمكن الاعتماد على حلول جاهزة قابلة للتخصيص:

| الحل | المميزات | الملاحظات |
|------|----------|-----------|
| **Odoo** | يجمع المتجر + ERP (مخزون، محاسبة، مشتريات) في نظام واحد | الأسرع لتغطية الـ ERP كاملاً |
| **Medusa.js** | Headless commerce مفتوح المصدر + قابل للتوسعة | يحتاج بناء واجهة مخصصة |
| **WooCommerce** | سريع وسهل، نظام إضافات ضخم | الـ ERP يحتاج إضافات منفصلة |

> **التوصية:** إن كان نظام ERP المتكامل أولوية قصوى وبسرعة → **Odoo**. إن كانت تجربة المستخدم المخصصة والتحكم الكامل أولوية → بناء مخصص بـ Next.js + NestJS.

## 4. البنية التحتية (Infrastructure)

| العنصر | الخيار المقترح |
|--------|----------------|
| الاستضافة | خادم سحابي (AWS / DigitalOcean / Hetzner) |
| الحاويات | Docker + Docker Compose |
| شهادة الأمان | SSL/TLS (Let's Encrypt) |
| النسخ الاحتياطي | نسخ يومي تلقائي لقاعدة البيانات |
| المراقبة | أدوات مراقبة الأداء والأخطاء (Sentry / Grafana) |
| CDN | لتسريع تحميل الصور والملفات الثابتة |

## 5. مكوّنات النظام الرئيسية

```
المتجر (Storefront)
 ├── الصفحة الرئيسية
 ├── صفحات التصنيفات
 ├── صفحة المنتج
 ├── السلة والدفع
 └── حساب العميل

لوحة الإدارة (Admin/ERP)
 ├── لوحة المعلومات (Dashboard)
 ├── إدارة المنتجات
 ├── إدارة المخزون
 ├── إدارة الطلبات
 ├── المشتريات والموردين
 ├── المحاسبة والتقارير
 ├── إدارة العملاء (CRM)
 └── المستخدمون والصلاحيات

الخدمات الخلفية (Services)
 ├── خدمة المصادقة
 ├── خدمة الدفع
 ├── خدمة الشحن
 ├── خدمة الإشعارات
 └── خدمة البحث
```

## 6. واجهات برمجية (APIs) رئيسية

| المجموعة | أمثلة على النقاط (Endpoints) |
|----------|------------------------------|
| المنتجات | عرض، تفاصيل، فلترة، بحث |
| السلة | إضافة، تعديل، حذف، عرض |
| الطلبات | إنشاء، تتبّع، إلغاء |
| المخزون | تحديث الكمية، الحجز، التنبيهات |
| المحاسبة | تقارير، فواتير |
| المصادقة | تسجيل، دخول، تجديد التوكن |

> كل واجهات الإدارة محمية بالمصادقة والصلاحيات (انظر ملف 10).
