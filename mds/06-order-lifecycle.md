# 06 — دورة حياة الطلب وترابطه مع المخزون

> هذا الملف يوضّح كيف ينتقل الطلب من الإنشاء حتى التسليم، وكيف يتزامن تلقائياً مع المخزون والمحاسبة.

---

## 1. التدفق الكامل للطلب

```
┌────────────────────────────────────────────────────────┐
│ 1. العميل يضيف منتجات للسلة                              │
│    → التحقق من توفّر الكمية لحظياً                       │
└───────────────────────────┬────────────────────────────┘
                            ▼
┌────────────────────────────────────────────────────────┐
│ 2. العميل يتمم الطلب (Checkout)                          │
│    → حجز الكمية مؤقتاً (reserved_quantity +)            │
│    → الحالة: "جديد"                                      │
└───────────────────────────┬────────────────────────────┘
                            ▼
┌────────────────────────────────────────────────────────┐
│ 3. تأكيد الطلب (تلقائي للدفع الإلكتروني / يدوي لـ COD)  │
│    → خصم نهائي من المخزون (stock - / reserved -)        │
│    → تسجيل حركة مخزون (إخراج)                            │
│    → إنشاء فاتورة + قيد محاسبي                           │
│    → الحالة: "مؤكد"                                      │
│    → إشعار للعميل                                        │
└───────────────────────────┬────────────────────────────┘
                            ▼
┌────────────────────────────────────────────────────────┐
│ 4. تجهيز الطلب                                           │
│    → تعبئة المنتجات وطباعة الفاتورة وبوليصة الشحن        │
│    → الحالة: "قيد التجهيز"                               │
└───────────────────────────┬────────────────────────────┘
                            ▼
┌────────────────────────────────────────────────────────┐
│ 5. الشحن                                                 │
│    → تعيين شركة الشحن والمندوب                           │
│    → رقم تتبّع                                            │
│    → الحالة: "مشحون"                                     │
│    → إشعار للعميل برقم التتبّع                            │
└───────────────────────────┬────────────────────────────┘
                            ▼
┌────────────────────────────────────────────────────────┐
│ 6. التسليم                                               │
│    → تأكيد الاستلام                                       │
│    → تحصيل المبلغ (لـ COD) → تحديث حالة الدفع           │
│    → الحالة: "مُسلّم"                                    │
│    → طلب تقييم من العميل                                 │
└────────────────────────────────────────────────────────┘
```

---

## 2. المسارات الاستثنائية

### الإلغاء (Cancellation)
```
طلب (جديد/مؤكد) → إلغاء
   → إعادة الكمية للمخزون (stock + / reserved -)
   → تسجيل حركة مخزون (تسوية)
   → استرداد المبلغ إن كان مدفوعاً
   → الحالة: "ملغي"
```

### الإرجاع (Return)
```
طلب (مُسلّم) → طلب إرجاع → موافقة
   → استلام المنتج المرتجع
   → إعادة الكمية للمخزون (حركة: مرتجع)
   → استرداد المبلغ (Refund)
   → الحالة: "مرتجع"
```

### انتهاء مهلة الحجز
```
طلب (جديد) لم يُكمَل خلال المدة المحددة
   → تحرير الكمية المحجوزة (reserved -)
   → إلغاء الطلب تلقائياً
```

---

## 3. منطق المخزون (Stock Logic)

### المعادلات الأساسية
```
الكمية المتاحة للبيع = stock_quantity - reserved_quantity

عند إنشاء الطلب:    reserved_quantity += كمية الطلب
عند تأكيد الطلب:    stock_quantity -= كمية الطلب
                    reserved_quantity -= كمية الطلب
عند الإلغاء:        reserved_quantity -= كمية الطلب
عند الإرجاع:        stock_quantity += كمية الطلب
```

### قواعد الحماية
- منع إتمام الطلب إذا كانت الكمية المتاحة أقل من المطلوبة.
- استخدام **قفل (Lock)** على السجل أثناء التحديث لمنع البيع المزدوج عند الطلبات المتزامنة.
- التحقق من التوفّر مرة أخرى لحظة التأكيد (وليس فقط عند الإضافة للسلة).

---

## 4. الترابط مع الوحدات الأخرى

| الحدث | التأثير على المخزون | التأثير على المحاسبة | التأثير على العميل |
|-------|---------------------|----------------------|--------------------|
| تأكيد طلب | خصم الكمية | تسجيل إيراد + فاتورة | تحديث سجل الشراء |
| إلغاء طلب | إعادة الكمية | إلغاء/استرداد | تحديث السجل |
| إرجاع | إعادة الكمية | استرداد مالي | تحديث السجل |
| استلام شراء | زيادة الكمية | تسجيل تكلفة | — |

## 5. الإشعارات المرتبطة بالحالات

| الحالة | الإشعار للعميل |
|--------|----------------|
| مؤكد | "تم تأكيد طلبك رقم #..." |
| قيد التجهيز | "جارٍ تجهيز طلبك" |
| مشحون | "تم شحن طلبك — رقم التتبّع: ..." |
| مُسلّم | "تم تسليم طلبك — نتمنى أن ينال إعجابك" + طلب تقييم |
| ملغي | "تم إلغاء طلبك" |

> الإشعارات تُرسل عبر البريد و/أو الرسائل النصية (SMS) و/أو إشعارات الموقع.
